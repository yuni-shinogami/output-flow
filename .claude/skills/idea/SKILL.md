# ネタ帳マネージャー

ネタは `output/<name>/idea.md` に保存する。1ネタ = 1ファイル。
ネタを記録したプロジェクトは、後から `/outline <name>` でそのままアウトラインワークフローに進める。

## コマンド

`$ARGUMENTS` を解析して処理を分岐する。

### `add` または引数なし → ネタ追加

1. ユーザーに以下を聞く（まとめて回答OK）：
   - **タイトル**（= プロジェクト名として使う）
   - **ひとことメモ** — 何を伝えたいか、なぜやりたいか等、思いついたことを自由に
   - **想定アウトプット形式**（任意） — LT / ブログ / 登壇 / 未定 など
   - **きっかけ・背景**（任意） — なぜ今これを思いついたか

2. `output/<タイトル>/idea.md` に保存する。フォーマットは [templates/idea.md](templates/idea.md) を参照。
   ディレクトリが未作成なら作成する。

3. 保存後、内容をユーザーに提示し確認する。

### `list` → ネタ一覧

1. `output/*/idea.md` を検索する。
2. 各ファイルの先頭から **タイトル** と **ステータス** を抽出する。
3. 一覧を表形式で表示する：

```
| # | タイトル | 形式 | ステータス |
```

ステータスは idea.md のみ存在 → 「ネタ」、outline/ も存在 → 「アウトライン作成中」 と表示する。

### `show <name>` → ネタ詳細表示

`output/<name>/idea.md` を読んで表示する。

### `brush <name>` → ネタのブラッシュアップ

アイデアを対話で深掘りする。構成は作らず、ネタの解像度を上げることだけに集中する。

1. `output/<name>/idea.md` を読み込み、現在の内容を提示する。
2. 以下のような問いかけで自由に対話する（順不同、状況に応じて選ぶ）：
   - 「これの何が面白いと思った？」
   - 「誰に一番届けたい？」
   - 「自分ならではの視点はどこにある？」
   - 「聞いた人/読んだ人にどうなってほしい？」
   - 「似たテーマの発表との違いは？」
   - 「今の時点で一番伝えたい一文は？」
3. 対話で出てきた内容を `idea.md` の **メモ欄** に追記する。
4. ユーザーが「終わり」「OK」などで対話を終えたら、追記した内容を提示して確認する。

**outline との違い**: outline は構造化された成果物（context.md, goal.md, outline.md）を作る。brush はあくまで idea.md 内のメモを育てるだけで、形式を整えない。

## 注意

- ネタはあくまでメモ。ユーザーに深掘りを強制しない。任意項目が空でも保存する。
- タイトルはそのままディレクトリ名になるため、ユーザーが指定した表現をそのまま使う。
