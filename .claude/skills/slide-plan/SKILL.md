# Phase 1: スライド設計

プロジェクト: **$ARGUMENTS**

`$ARGUMENTS` 未指定なら案内して終了。
まず `output/$ARGUMENTS/outline/context.md`、`output/$ARGUMENTS/outline/goal.md`、`output/$ARGUMENTS/outline/outline.md` を読み込む。いずれか不在なら該当フェーズを案内。

## ユーザーに聞くこと

アウトラインの内容（特に各章の時間配分）を踏まえて、以下を聞く:

1. **最終出力形式** — Marpのみで完結（カスタムテーマ使用） / Canvaで仕上げ / その他
   - Marpのみの場合: カスタムテーマ（`.claude/skills/slide/themes/starter/starter-theme.css`）を使用。`.claude/skills/slide/themes/starter/docs/patterns.md` のパターンカタログ参照
   - Canva仕上げの場合: Canvaテンプレートファイルの有無・パスを確認
2. **ビジュアル方針** — テキスト中心 / ビジュアル重視 / バランス型
3. **スライド密度** — 1トピック1スライド / コンパクトにまとめる
5. **掴み（Hook）の方針** — どんな切り口で聴衆の興味を引くか（問いかけ / 衝撃的な事実 / あるある共感 / etc.）
6. **お品書き（Agenda）の3項目** — アウトラインの章構成をもとに、お品書きとして提示する3つのトピック名を決める
7. **画像・SVGの方針** — ユーザー提供画像の有無（ファイル名・配置箇所）、Claude生成SVG図解の希望箇所（フロー図・比較チャート等）
8. **特別な制約**（任意） — コードブロックの有無、会社テンプレートの有無
9. **構成パターン** — 問題解決型 / 体験談型 / PREP型（結論先出し）。context.mdの場やテーマに応じて推奨を提示
10. **ブリッジスライド** — セクション転換にオレンジグラデーション全面+白太字の1メッセージスライドを使うか（推奨: 使う）
11. **段階的ビルドアップ** — 図解や概念説明で段階的に情報を追加する手法を使うか
12. **クロージング後の構成** — おまけ（軽い笑い/失敗談） / 宣伝（活動紹介・QRコード等） / なし

## 回答後の処理

回答とアウトラインの内容を統合し、`output/$ARGUMENTS/slide/slide-plan.md` に保存する。フォーマットは [templates/slide-plan.md](templates/slide-plan.md) を参照。

slide-plan.mdには以下を含める:
- 最終出力形式（Marp/Canva/その他）とCanvaテンプレート情報
- ビジュアル方針（テーマ、密度、スタイル）
- プレゼン構造（掴みの方針・内容案）
- お品書き（3トピックとアウトライン章の対応）
- パターン選定（各スライド→推奨レイアウトパターンのマッピング）
- アウトラインの各章→スライド枚数・時間配分の対応表（固定スライド含む）
- 注意スライド（特に工夫が必要なスライド）
- 構成パターン（問題解決型/体験談型/PREP型）
- ブリッジスライド戦略（配置箇所と頻度）
- ビルドアップ戦略（使用箇所）
- クロージング後の構成（おまけ/宣伝/なし）
- 制約事項

パターン選定では `.claude/skills/slide/themes/starter/docs/patterns.md` のカタログを参照し、各スライドに最適なレイアウトパターン（`_class:`）を割り当てる。

保存後、整理した内容をユーザーに提示し「この方針でPhase 1を完了してよいか？」と確認する。

**重要: ユーザーが明示的に同意するまで次フェーズに進んではならない。追加・修正の要望があれば反映してから再度確認する。**

同意が得られたら「Phase 1 完了」と伝え、Skillツールで `slide-draft` を引数 `$ARGUMENTS` で起動する。
