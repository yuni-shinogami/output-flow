# レビュー用サブエージェントプロンプト

以下のテンプレートの `{goal}` `{write_plan}` `{draft}` に実際の内容を埋め込んで使う。

## 1. ゴールとの整合性チェック

```
以下のゴール定義と下書きを読み、「ゴールとの整合性」を点検してください。

【観点】
- ゴールで定めたメッセージが読者に伝わるか
- 受け手に期待する行動/変化につながる内容になっているか
- 目的から逸れた内容、不足している内容はないか

【ゴール定義】
{goal}

【下書き】
{draft}

問題点は具体的な箇所を引用して指摘し、改善案を出す。問題なければ「問題なし」と報告。
```

## 2. 読みやすさ・文体の一貫性チェック

```
以下の執筆プランと下書きを読み、「読みやすさと文体の一貫性」を点検してください。

【観点】
- 執筆プランで定めたトーン・人称・語尾が一貫しているか
- 冗長な表現、回りくどい箇所はないか
- 文のリズム（長い文が連続していないか、単調でないか）
- 読者が引っかかりそうな不自然な表現はないか

【執筆プラン】
{write_plan}

【下書き】
{draft}

問題点は具体的な箇所を引用して指摘し、改善案を出す。問題なければ「問題なし」と報告。
```

## 3. 独自価値の表現チェック（最重要）

```
以下の下書きを読み、「独自価値の表現」を点検してください。これは最も重要なチェックです。

【観点】
- 自分の経験・体験に基づく内容が具体的に描写されているか
- 一般論や誰でも書ける内容に埋もれていないか
- 「この人だから書ける」価値が出ているか
- 経験を語る箇所の臨場感・具体性は十分か

【下書き】
{draft}

各セクションについて、独自の経験に基づく部分と一般的な記述を分類する。
一般的な記述が目立つ箇所には、どんな経験やエピソードを追加・深掘りすれば独自価値が出るか具体的に提案する。
```

## 4. 構成・流れチェック

```

- セクション間のつなぎは自然か
- テンポは適切か（緩急があるか）
- 結びは余韻を残せるか

【下書き】
{draft}

```

## 5. LLM表現パターンチェック

```
以下の下書きを読み、「LLM特有の表現パターン」を点検してください。

【観点】
以下の11カテゴリに該当するパターンが繰り返し・集中して現れていないかチェックする。1箇所の使用は許容し、繰り返しを問題とする（カテゴリ6, 11は1回でも対象）。

1. 曖昧な断言回避（ヘッジング）: 「〜と言えるでしょう」「〜かもしれません」「〜と考えられます」の多用
2. 過剰な要約フレーズ: 「重要なのは〜です」「つまり〜ということです」でセクション毎に要約を繰り返す
3. 過剰な箇条書き・並列構造: 常に3項目の箇条書き、同一文法構造の繰り返し
4. 定型的な接続表現: 「まず→次に→さらに→最後に」のような機械的な順序接続の連続
5. 過剰な強調: 「非常に」「極めて」「とても」「まさに」の多用（500文字あたり2回以上）
6. 結びの定型表現: 「いかがでしたでしょうか」「少しでも参考になれば幸いです」（1回でも対象）
7. 不自然な丁寧さのエスカレーション: 文末敬語が後半に向かって過剰に丁寧になる
8. 抽象的で具体例のない記述: 「〜は重要です」「〜が求められます」のみで具体例が続かない
9. 万能型まとめパラグラフ: キーワードを入れ替えれば別記事にも使い回せる結論
10. ダッシュによる挿入・転換: 「——つまり」「これは——言い換えれば——」のように英語圏のダッシュ用法を日本語に持ち込む
11. ダブルクォートによる強調: 「"重要"」「"楽しい"」のように""で強調する。日本語では「」を使う（1回でも対象）

【下書き】
{draft}

カテゴリごとに該当箇所を引用し、深刻度（高/中/低）をつけて報告する。修正案はbefore/after形式で出す。該当なしのカテゴリは省略。全て問題なければ「問題なし」と報告。
```
