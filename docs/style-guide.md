## カラーパレット

### プライマリカラー
| 名前 | HEX | 用途 |
|------|-----|------|
| Orange | `#F5A623` | メインアクセント、ヘッダーバー、箇条書きマーカー、ボーダー |
| Orange Dark | `#F09819` | グラデーション始点 |
| Orange Light | `#F7CE68` | グラデーション終点 |

### グラデーション
- **背景グラデーション**: `linear-gradient(135deg, #F09819 0%, #F7CE68 100%)`
- 使用箇所: cover, section-box, section-line, bridge, result, summary
- **実スライドでの使用頻度:** プレゼンあたり3〜8回（ブリッジ含む）

### テキストカラー
| 名前 | HEX | 用途 |
|------|-----|------|
| Black | `#333333` | 本文テキスト |
| White | `#ffffff` | グラデーション背景上のテキスト |
| Gray | `#e0e0e0` | 罫線、グリッド背景、ツリーの縦線 |

### 制約（実スライドから確認済み）
- 1スライドの背景は「白」か「グラデーション」の2択（例外なし）
- アクセント色はオレンジ系のみ使用 — 他の色は一切使わない
- テキスト色は `#333333`（本文）と `#ffffff`（グラデーション背景上）のみ。キャプションやサブテキストも `#333333` を使用する（`#666` 等の中間グレーは使用禁止）
- グラデーション背景の上のテキストは必ず白
- 会社紹介スライド（coconala等）は別ブランドカラーを使うが、個人スライドには混ぜない
- グレー背景はデータ/グラフ表示時のみ例外的に使用可

## タイポグラフィ

### フォントスタック
```
"Noto Sans JP", "Hiragino Kaku Gothic ProN", "Yu Gothic", sans-serif
```

### サイズ階層（em基準）
| レベル | サイズ | weight | 用途 |
|--------|--------|--------|------|
| 特大 | 4em | 900 | title の h1 |
| 大 | 3em〜3.2em | 900 | cover h1, emphasis h1, summary h1 |
| 中大 | 2.4em〜2.8em | 900 | section h1, result h1, emphasis-detail h1 |
| 中 | 1.6em | 700 | コンテンツ h2 |
| 標準 | 1.3em | 400/700 | 箇条書き項目 |
| 小 | 1.1em〜1.2em | 400 | 補助テキスト、サブタイトル |
| 極小 | 0.85em〜0.95em | 400 | ネスト項目、プロフィール詳細 |

### ジャンプ率（視覚的階層の強さ）
重要度に応じてフォントサイズの差を大きくする。全要素が同じサイズだと階層が伝わらない。

| レベル | サイズ | 前レベル比 | 用途 |
|--------|--------|-----------|------|
| 特大 | 4em | — | title の h1 |
| 大 | 3〜3.2em | ×0.8 | cover, emphasis, summary |
| 中大 | 2.4〜2.8em | ×0.8 | section, result, emphasis-detail |
| 中 | 1.6em | ×0.6 | h2（見出し） |
| 標準 | 1.3em | ×0.8 | 箇条書き |
| 小 | 1.1em | ×0.85 | サブテキスト |
| 極小 | 0.85〜0.95em | ×0.8 | ネスト項目、注釈 |

推奨: 見出しと本文の最小サイズ比は 1.2倍以上

### 行間
- ベース行間: `1.5`（section デフォルト）
- 箇条書き: `1.4`（padding 12px と合わせて適切な間隔）
- 見出し（h1, h2）: 指定なし → ベース `1.5` を継承
- プロフィール詳細: `1.5`
- プロジェクター投影を考慮し、一般文書の推奨値（1.2〜1.3）より広めに設定

### 改行・一行長ルール
- 1行が画面端から端まで続かないようにする（padding で自動制御）
- 日本語テキストは1行あたり 25〜35文字 が読みやすさの目安
- h1 が2行に渡る場合、意味のまとまりで改行位置を調整する
- 改行位置の注意:
  - 単語や助詞の途中で切れないようにする
  - 1〜2文字だけ次の行にはみ出す場合は文言を調整する
  - Marp では `<br>` で明示的に改行位置を制御できる

### 装飾
- **太字イタリック**: コンテンツ系のh2（`font-weight: 700; font-style: italic`）
- **ストローク**: title の h1（`-webkit-text-stroke: 2px var(--color-orange-dark)`）
- **太字イタリック**: summary h1

### 孤立文字（orphan character）の防止
- 1〜2文字だけ次の行にはみ出す「孤立文字」はスライドの見栄えを大きく損なう
- 原因: テキスト幅がコンテンツ幅をわずかに超え、最後の数文字が2行目に落ちる
- font-weight: 900 の太字は通常より描画幅が広く、文字数計算だけでは判定しにくい
- **対策**: 大きいフォントのh1は `<br>` で意味のまとまりごとに改行位置を明示する
- **パターン別の1行あたりの文字数上限**（全角文字基準、太字による変動あり）:

| パターン | font-size | 1行の目安 | 備考 |
|---------|-----------|----------|------|
| title h1 | 4em | ~9文字 | 2〜3行を前提に `<br>` 必須 |
| cover h1 | 3em | ~14文字 | 超える場合 `<br>` |
| emphasis h1 | 3.2em | ~11文字 | 超える場合 `<br>` |
| bridge h1 | 2.8em | ~13文字 | 12文字でも溢れることあり |
| emphasis-detail h1 | 2.4em | ~15文字 | 超える場合 `<br>` |
| section-box h1 | 2.8em | ~13文字 | 白枠内 |
| section-line h1 | 2.8em | ~13文字 | |
| summary ol li | 1.3em | ~18文字 | 白枠内のため狭い |
| content h2 | 1.6em | ~25文字 | 余裕あり |
| content li | 1.3em | ~30文字 | 25〜35文字目安 |

### パターン容量制限
- 各パターンの容量の目安（h1行数、箇条書き項目数の上限等、1行あたりの文字数上限）は `docs/patterns.md` の各パターン説明を参照

### 実スライドで確認された使い分け
- 強調したいフレーズは**フォントサイズを極端に大きく**する（3XL以上）
- 1スライドに複数のフォントサイズを混在させて**視覚的階層**を作る
- ブリッジスライドのテキストは2-3行が上限（それ以上は詰め込みすぎ）

## スペーシング

### 余白の原則
- スライドの上下左右に十分な余白を確保する（本文1〜2文字分が目安）
- 現在の設定: 標準 `60px 80px` = 1280px幅の約6%（上下）/ 約12%（左右）
- 囲み枠（白枠ボックス、番号リスト枠）の内部にも余白を設ける
- タイトル（h2）と本文の間に `24px` 以上の間隔を確保する
- 要素を端いっぱいまで配置しない

### セクション padding
- 標準: `60px 80px`（section デフォルト）
- グラデーション系: `0`〜`80px 120px`（中央配置のため）
- summary: `50px 100px`

### 要素間隔
- h2 → コンテンツ: `margin-bottom: 24px`
- 箇条書き項目間: `padding: 12px 0`
- ネスト項目間: `padding: 6px 0`

### 配置ルール
- 左端を揃える: content 系パターンのh2、箇条書きは全て左端 80px 起点
- 関連する情報はグループ化して近くに配置し、無関係な情報は間隔を空ける
- 間隔は 8px の倍数を基本とする（8, 16, 24, 40, 60, 80px）
- 画像+テキストの gap: 40px

## 装飾パターン

### ヘッダーバー
- コンテンツ系スライド（content, content-tree, emphasis, emphasis-detail, conclusion-bullets）
- オレンジ背景 + 白文字
- `width: fit-content`（テキスト幅に合わせる）
- スライド左上に配置（`margin: -60px -80px 40px`）

### 白枠ボックス
- section-box, result で使用
- `border: 4px solid white; border-radius: 12px; padding: 40px 80px`

### アンダーライン
- section-line で使用
- `border-bottom: 4px solid white`

### 箇条書きマーカー
- オレンジ色の `▶`（`content: "▶"; color: var(--color-orange)`）
- 位置: `position: absolute; left: 0`
- 実スライドで最も頻出する装飾要素

### blockquote
- emphasis-detail で使用するまとめ用引用ブロック
- テキスト色は `#333333`（`var(--color-black)`）
- パディング/マージンなし（コンパクト表示）

### ツリー構造の縦線
- `border-left: 3px solid var(--color-gray)`

### グリッド背景
- title スライド専用
- 50px 間隔の薄いグレー線

### オレンジ丸角ピルラベル（実スライド由来）
- 「まえがき」「おまけ」「具体例②」のようなラベルバッジ
- オレンジ背景 + 白文字 + 丸角 + 中央寄り配置
- section-box の emphasis パターンの亜種として使用可能

### オレンジボーダー枠（実スライド由来）
- QRコードや重要情報を囲む枠線: `border: 4px solid #F5A623; border-radius: 12px`
- 強調したいコンテンツブロックに使用

### キャラクターイラスト（実スライド由来）
- VRChatアバター（ピンク髪・黒髪）を吹き出しとセットで使用
- お品書きスライドやQ&Aスライドに配置
- 画像ファイルとして用意し、HTML `<img>` で配置

## SVG / 画像ガイドライン

- SVG生成の詳細は `docs/svg-guide.md` を参照
- カラー制約: SVG内の色もオレンジ系のみ（スライドと同じ制約）
- サイズ: viewBoxはMarp 16:9（1280x720）に合わせた規約
- 配置: 画像レイアウトクラス（`img-text-right` 等）を使用（`docs/patterns.md` 参照）
- インラインSVG: 20行以下の小さな図。直接 `slides.md` 内に記述
- 外部SVG: 複雑な図。`work/<project>/images/*.svg` に保存し `<img>` で参照

## NGパターン

- グラデーション背景に黒テキスト（コントラスト不足）
- 白背景にオレンジ以外のアクセント色（全プレゼンで厳守されている）
- 1スライドに3色以上のアクセント色
- ヘッダーバーをグラデーション背景スライドに配置
- 箇条書き6項目以上（5項目以内が目安、実スライドでは3-5項目が標準）
- `font-size` を px 指定（em を使うこと — Marp のスケーリングに追従するため）
- ブリッジスライドに4行以上のテキスト（1-3行が上限）
- 1スライドに複数の主張を詰め込む（1スライド1メッセージが基本）
- SVGにパレット外の色を使用（オレンジ系・グレー・白・黒のみ）
- 外部SVGファイルに意味のないファイル名（`image1.svg` 等は不可、`flow-deploy.svg` のように内容を示す）
- インラインSVGが20行超（外部ファイル化すべき）

## TailwindCSS ユーティリティクラスの使い方

カスタムテーマクラス（`_class: cover` 等）で基本レイアウトを決め、TailwindCSS ユーティリティで微調整する。

### 推奨する使い方
- `grid`, `flex` — カスタムレイアウト内の要素配置
- `bg-brand-orange`, `text-brand-orange` — テーマカラーの適用
- `gap-*`, `p-*`, `m-*` — スペーシング調整
- `text-slide-*` — フォントサイズ微調整
- `rounded-*`, `border-*` — 装飾

### 避けるべき使い方
- `bg-*`（テーマカラー以外） — カラーパレット外の色を使わない
- `text-*`（px ベース） — em ベースの `text-slide-*` を使う
- `font-*` — カスタムテーマの font-family / weight を上書きしない
