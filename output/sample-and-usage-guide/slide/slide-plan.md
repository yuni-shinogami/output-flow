# スライド設計プラン

## 最終出力形式
- 形式: Marpのみで完結
- Canvaテンプレート: なし
- 備考: output-flowのサンプルプロジェクト用。スライドワークフローの実例として機能する

## ビジュアル方針
- テーマ: starter-theme（スターターテーマ）
- スタイル: テキスト中心
- 密度: 1トピック1スライド
- ブリッジスライド: 使用する

## プレゼン構造
- 構成パターン: 問題解決型
- 掴みの方針: あるある共感
- 掴みの内容案: 「ネタはあるのにブログが書けない」— エンジニアの誰もが経験する悩みで共感を得る
- ビルドアップ箇所: ワークフロー全体像（/outline → /write を先に見せ、次に /slide を追加して3つ揃える。2枚構成）

## お品書き（3トピック）
1. 「書けない」の正体 — 対応アウトライン章: 1章
2. output-flow のアプローチ — 対応アウトライン章: 2章
3. 使ってみよう — 対応アウトライン章: 3章

## パターン選定

`.claude/skills/slide/themes/starter/docs/patterns.md` のカタログから各スライドに最適なレイアウトパターンを割り当てる。

| スライド | パターン（`_class:`） | 理由 |
|:---------|:---------------------|:-----|
| 表紙 | `cover` | グラデーション中央テキスト |
| 掴み | `emphasis` | 共感フレーズを大太字で |
| 自己紹介 | `self-intro` | 左右2カラムプロフィール |
| ブリッジ | `bridge` | 全画面1メッセージ |
| お品書き・転換お品書き | `content` | 箇条書き3項目 |
| 問題提起（手順がない） | `emphasis-detail` | 大太字+補足箇条書き |
| 対話で言語化 | `content` | 箇条書きで説明 |
| ワークフロー全体像 | `content` | ビルドアップ2枚 |
| 確認しながら進む設計 | `emphasis-detail` | 特徴を強調+補足 |
| セットアップ | `content` | コードブロック+説明 |
| メタ的種明かし | `emphasis` | インパクトフレーズ |
| まとめ | `summary` | 番号リスト |
| クロージング | `bridge` | 締めの1メッセージ |
