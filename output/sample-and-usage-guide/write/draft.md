## 「書きたいのに書けない」問題

カンファレンスに参加して刺激を受けた。新しいライブラリを試して面白い知見を得た。LTのネタになりそうな経験をした。——なのに、ブログが書けない。

エンジニアなら一度はこの状態に陥ったことがあるのではないでしょうか。

自分もそうでした。書きたいことはあるのに、エディタを開くと手が止まる。「何から書けばいいんだろう」で30分が過ぎ、結局下書きフォルダにタイトルだけのファイルが増えていく。

これ、文章力の問題ではありませんでした。**手順**の問題だったんです。

いきなり本文を書こうとするのは、レシピを見ずに料理を始めるようなものです。材料（ネタ）はあるのに、手順（構成）がないから手が止まる。自分はこの「手順」を対話で作るツールを作ることで、ようやく書けるようになりました。

## output-flow というアプローチ

output-flow は、Claude Code のスキル機能を使った対話型のアウトプット作成ツールです。

核となるアイデアはシンプルです。**自分の中にある素材を、AIとの対話で引き出して構造化する**。

### 2つのワークフロー

```
/outline（骨格づくり）──→ /write（記事執筆）
```

まず `/outline` でアウトプットの骨格を作ります。「誰に向けて」「何を伝えたくて」「どういう順序で話すか」を対話で整理していきます。

骨格ができたら `/write` で本文を書きます。文体やトーンを決めてから、セクションごとに執筆し、最後にレビューして仕上げます。

### 「確認しながら進む」設計

output-flow の各フェーズには必ず確認ステップがあります。例えば、コンテキストを整理した後には「この内容で Phase 1 を完了してよいか？」と聞かれます。

AIが勝手に先に進んだり、意図しない方向に暴走したりしません。対話のペースは常にあなたが握っています。

## 実際に使ってみる

### セットアップ

```bash
git clone https://github.com/yuni-shinogami/output-flow.git
cd output-flow
claude
```

これだけです。Claude Code がインストール済みなら、clone して `claude` を起動するだけでスキルが使えます。

ちなみに、この記事自体も output-flow を使って書いています。以降の説明は、まさにこの記事を作った過程そのものです。

### /outline の流れ

```
/outline sample-and-usage-guide
```

4つのフェーズを対話で進めます。

1. **コンテキスト設定**: どこで公開するか、誰が読むか、制約は何か
2. **ゴール設定**: 読者にどうなってほしいか、自分のスタンスは何か
3. **構成作成**: 素材を出して、章・項に整理する
4. **レビュー**: 4つの観点（整合性・流れ・ロジック・独自価値）で点検

特に面白いのはゴール設定の「語りのスタンス」です。「結論を届けたい」「一緒に考えたい」「体験を共有したい」から選ぶことで、構成パターンが変わります。この記事では「体験を共有したい」を選びました。

### /write の流れ

```
/write sample-and-usage-guide
```

3つのフェーズで本文を仕上げます。

1. **執筆準備**: 文体・トーン・方針を決める
2. **本文執筆**: セクション単位で書いて、都度確認
3. **推敲**: 4観点でレビューし、対話しながら改善

### この記事自体がサンプルです

お気づきかもしれませんが、ここまで読んできたこの記事は、output-flow の `output/sample-and-usage-guide/` に格納されたサンプルプロジェクトです。リポジトリを clone すれば、context.md → goal.md → outline.md → write-plan.md → draft.md → article.md と、各フェーズの成果物を実際に読むことができます。

## おわりに

あなたの下書きフォルダに眠っている「タイトルだけのファイル」を1つ思い浮かべてください。

そのネタを `/outline` に投げてみてください。対話しているうちに、自分でも気づいていなかった「本当に書きたかったこと」が見えてくるかもしれません。

```bash
git clone https://github.com/yuni-shinogami/output-flow.git
cd output-flow
claude
# そして...
/outline あなたのネタ
```
