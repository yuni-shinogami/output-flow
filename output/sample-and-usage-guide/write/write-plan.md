# 執筆プラン

## 文体方針
- トーン: カジュアル（技術ブログらしい親しみやすさ）
- 人称: 自分
- 語尾: です・ます
- 参考スタイル: Zenn のトレンド記事のようなテンポ感

## 構成対応表

| アウトライン章 | 記事セクション | 想定ボリューム |
|:---:|:---:|:---:|
| 1. 「書きたいのに書けない」問題 | 導入 — 共感パート | 軽め |
| 2. output-flow というアプローチ | ツール紹介 — 仕組みの説明 | 標準 |
| 3. 実際に使ってみる | 使い方ガイド — 実例ベース | 厚め |
| 4. おわりに | クロージング — 行動の呼びかけ | 軽め |

## 執筆上の注意点
- 独自価値を活かすポイント: 「自分自身がこのツールで書けるようになった」体験を随所に織り込む。この記事自体がサンプルであるメタ構造を活用する。
- 避けるべきこと: 機能の羅列にならないこと。「なぜこの設計にしたか」の意図を伝える。
- その他: コードブロックでコマンド例を示し、読者がすぐ試せるようにする。

## 制約
- 文字数目安: 3,000〜5,000字
- フォーマット要件: Markdown。見出しは h2/h3 を使用。コマンド例はコードブロックで囲む。
