# アウトライン

## タイトル（仮）
Claude Codeで「書けない」を解消する — output-flow で記事執筆を対話型ワークフローにした話

## メタ情報
- コンテキスト: 技術ブログ向け、Claude Code ユーザーのエンジニアが読者
- ゴール: output-flow を試してもらう、対話で書く体験の解像度を上げる
- スタンス: 体験を共有したい

## ストーリーライン
体験の提示（書けない問題） → 解決アプローチ（対話型ワークフロー） → 実際の使い方 → 読者への問いかけ

## 構成

### 1. 「書きたいのに書けない」問題
- 1.1 ネタはあるのに手が止まる: エンジニアあるある。カンファレンスに行った、新しい技術を試した、でもブログが書けない。
- 1.2 原因は「構成力」ではなく「手順」: いきなり書こうとするから止まる。料理でレシピなしに作り始めるようなもの。

### 2. output-flow というアプローチ
- 2.1 対話で思考を引き出す: 自分の中にある素材を、AIとの対話で言語化・構造化する。
- 2.2 2つのワークフロー: `/outline` で骨格を作り、`/write` で本文を書く。全体像を図で示す。
- 2.3 「確認しながら進む」設計: 各フェーズでユーザーの同意を取る。AIが暴走しない安心感。

### 3. 実際に使ってみる
- 3.1 セットアップ: git clone して claude を起動するだけ。
- 3.2 /outline の流れ: コンテキスト → ゴール → 構成 → レビューの4フェーズを実例で紹介。
- 3.3 /write の流れ: 執筆準備 → 本文 → 推敲の3フェーズを紹介。
- 3.4 まとめ: 実はこの記事自体が output-flow で作られている（メタ的な種明かし）。

### 4. おわりに
- 4.1 読者への問いかけ: あなたの「書けないネタ」を1つ思い浮かべて、`/outline` に投げてみてほしい。

## 素材メモ
- この記事自体が output-flow で作られたサンプルであることを活用する
- セットアップが簡単（clone するだけ）な点を強調する
